API Reference

Password Strength Analyzer — REST API v2.1

Base URL

http://127.0.0.1:5000

Security

  • All endpoints enforce Content-Type: application/json
  • Rate limit: 30 requests / 60 seconds per IP
  • Passwords are never logged, stored, or persisted
  • All responses include strict security headers (CSP, X-Frame-Options, etc.)
  • Maximum password input length: 128 characters
  • Spaces and whitespace characters are NOT allowed in passwords and will result in "weak" rating
POST /analyze

Analyze a password and return full strength analysis including criteria, score, entropy, patterns, crack times, and optional HIBP check.

Request Body

{ "password": "string", // required, max 128 chars, no spaces allowed "check_hibp": false // optional bool — enables HaveIBeenPwned check }

Response

{ "score": 85, // 0–100 overall strength score "entropy": 72.4, // bits of entropy "level": "strong", // none | weak | fair | strong | vstrong "level_label": "Strong", // human-readable label "length": 16, // character count "is_common": false, // matched common password list "criteria": { "meets_min": true, // >= 8 characters "meets_rec": true, // >= 12 characters "has_upper": true, "has_lower": true, "has_number": true, "has_symbol": true // NOTE: spaces are NOT considered symbols }, "patterns": [ { "type": "spaces_detected", "description": "Spaces or whitespace characters detected — not allowed in passwords", "severity": "high" } ], "crack_times": { "online_throttled": { "label": "Online attack (throttled)", "display": "3 years", "seconds": 9.4e7 }, "online_unthrottled": { "label": "Online attack (unthrottled)", "display": "4 days", "seconds": 3.4e5 }, "offline_gpu": { "label": "Offline attack (single GPU)", "display": "2 minutes", "seconds": 114 }, "offline_botnet": { "label": "Offline attack (botnet)", "display": "1 seconds", "seconds": 1.1 } }, "zxcvbn": { "score": 3, // 0–4 (null if zxcvbn not installed) "guesses": 1234567, "crack_times": { ... }, "feedback": { "warning": "string | null", "suggestions": ["string", ...] }, "sequence": [{ "pattern": "dictionary", "token": "pass" }] }, "hibp": { "pwned": false, // null if check_hibp was false "count": 0, "error": null }, "feedback": [ { "type": "tip", "icon": "💡", "text": "Add uppercase letters..." }, { "type": "warn", "icon": "🚨", "text": "Spaces not allowed..." }, { "type": "good", "icon": "✅", "text": "Strong length..." } ], "zxcvbn_available": true }

Example Request

curl -X POST http://127.0.0.1:5000/analyze \ -H "Content-Type: application/json" \ -d '{"password": "MyStr0ng!Pass", "check_hibp": true}'
POST /report

Generate and download a plain-text security report. The password itself is never written into the report.

Request Body

{ "password": "string" // required, max 128 chars }

Response

Returns text/plain file download: password_security_report.txt

Example Request

curl -X POST http://127.0.0.1:5000/report \ -H "Content-Type: application/json" \ -d '{"password": "MyStr0ng!Pass"}' \ --output report.txt

Rate Limiting

The API uses a sliding-window rate limiter. In production with Redis, limits are shared across all workers. In development, an in-memory fallback is used.

HaveIBeenPwned Integration

When check_hibp: true is passed, the API checks the password against the HIBP Pwned Passwords database using k-anonymity:

  1. SHA-1 hash of the password is computed locally
  2. Only the first 5 characters of the hash are sent to HIBP
  3. HIBP returns all hashes matching that prefix
  4. The matching is done locally — the full hash and password never leave the server

Password Validation Rules (v2.1)

  • Spaces/Whitespace: NOT ALLOWED — passwords with spaces will always be rated "weak" regardless of other criteria
  • Length: Minimum 8 characters, recommended 12+
  • Character Sets: Uppercase, lowercase, numbers, and special characters are recommended but not required individually
  • Special Characters: @#!$% and other non-alphanumeric characters (excluding whitespace) are recognized and awarded points
  • Common Passwords: Detected against a list of 100+ common/breached passwords and variants
  • Patterns: Keyboard walks (qwerty), repeating chars, sequences, dictionary words, year patterns all trigger warnings
